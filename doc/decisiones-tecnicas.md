# Uso de Lombok en el proyecto
¿Qué es Lombok? 
- Lombok es una librería de Java que permite reducir el código repetitivo (boilerplate) en clases, especialmente en entidades, DTOs y modelos.

### Este proyecto no utiliza Lombok.

Razones:
* El desafío debe ser fácilmente ejecutable sin configuraciones adicionales en el IDE del evaluador.
* Lombok requiere plugins específicos para funcionar en IDEs; sin ellos el proyecto no compila.
* Las entidades JPA (Java Persistence API) funcionan de manera más segura usando constructores y métodos explícitos.
* Al tratarse de un proyecto técnico evaluativo, se prioriza claridad, transparencia del código y compatibilidad total.

- Conclusión: Se eligió Java puro para evitar dependencias innecesarias y garantizar total compatibilidad.
